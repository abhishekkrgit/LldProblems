Functional Requirement
1. Game will be played on 3*3 board.
2. Each player will get alternate chance with marker 'X' and 'O'
3. Game should be able to declate draw if all cells are fullfill.
4. Game should be able to declare winner.
5. Game should reject invalid move;
6. There will be central ScoreCard which will stores wins,loses and draws of player


Non-Functional Requirement
1. System should follow object oriented design
2. System should be modular and extensible so that in future it can support new features like 4*4 boards, Ai Opponent, Move History;
3. Game logic should be easily testable
